# Glossary

Editor vocabulary you'll encounter throughout this project and in discussions with the expert agents.

## Editing / assembly

- **Rough cut** — first assembly of clips, timing approximately right, nothing polished
- **Fine cut** — timing locked, each shot duration optimized
- **Picture lock** — cut is final, no more timing changes; moves to color and audio
- **Assembly edit** — just placing clips in order, no trimming
- **Rough / fine / final cut** — progression of polish stages
- **Rough bin** — loose collection of possibly-usable clips
- **Selects** — clips you've chosen as candidates for use
- **Stringout** — all selects laid end-to-end without trimming, for review
- **Handles** — extra frames before/after a cut point, for trim flexibility
- **Match cut** — a cut where one element carries across the boundary
- **Jump cut** — a cut within the same shot/subject, time-jumping
- **L-cut / J-cut** — audio bleeds before or after the visual
- **Cut on action** — timing a cut during a motion, hiding the cut
- **In point / out point** — start and end of a clip segment
- **Trim** — adjusting a clip's in or out point
- **Ripple edit** — trim that shifts everything downstream
- **Overwrite** — placing a clip over existing material
- **Insert** — placing a clip between existing material, pushing things later

## Audio

- **LUFS** — Loudness Units Full Scale, perceptual loudness measurement
- **dB / dBFS** — decibel, digital full scale
- **dBTP** — decibel True Peak (accounts for inter-sample peaks)
- **Integrated LUFS** — average loudness across entire piece
- **Peak** — highest instantaneous level
- **Headroom** — gap between peak and 0 dBFS (ceiling)
- **Ducking** — automatically reducing one audio track when another plays (sidechain)
- **Sidechain** — compressor triggered by external signal (usually the vocal or kick)
- **Crossfade** — audio equivalent of dissolve, smoothing transition
- **Stem** — a mix of one type of audio (music stem, dialogue stem, SFX stem)
- **Mix bus / master bus** — final combined output
- **EQ** — equalization, frequency-specific volume adjustment
- **High-pass / low-pass** — filter removing low or high frequencies
- **Compression** — reducing dynamic range
- **Limiting** — aggressive compression, prevents exceeding ceiling
- **Foley** — re-recorded sync sound effects (footsteps, cloth, etc.)
- **Risers / downlifters** — sweeping SFX for building tension or releasing it

## Camera / cinematography

- **ELS** — Extreme Long Shot (character tiny in frame)
- **LS** — Long Shot (full body, environment visible)
- **MS** — Medium Shot (waist-up)
- **MCU** — Medium Close-Up (chest-up)
- **CU** — Close-Up (head/face dominant)
- **ECU** — Extreme Close-Up (eye, mouth, detail)
- **OTS** — Over-the-Shoulder
- **Two-shot** — two characters in frame
- **Dutch angle / Dutch tilt** — camera rotated, horizon skewed
- **POV** — Point of View shot
- **Insert** — close-up on an object/detail
- **Establishing shot** — wide shot setting location/context
- **Master shot** — wide shot of the full scene, usually the assembly anchor
- **Dolly / track / truck** — moving camera horizontally
- **Crane / jib** — vertical camera movement
- **Pan / tilt** — rotating camera (horizontal / vertical)
- **Zoom** — focal length change (optical or digital)
- **Push-in / pull-out** — camera moving toward or away from subject
- **Handheld** — unstabilized camera (intentionally shaky)
- **Rack focus** — shifting focus between depth planes mid-shot

## Color

- **LUT** — Look-Up Table (color transform file, .cube extension typical)
- **Primary** — initial color correction (exposure, WB)
- **Secondary** — targeted correction (specific color, specific area)
- **White balance (WB)** — setting neutral color temperature
- **Color temperature** — warm (orange) to cool (blue) axis, measured in Kelvin (K)
- **Tint** — green-to-magenta axis
- **Saturation** — color intensity
- **Luminance** — brightness independent of color
- **Chrominance** — color independent of brightness
- **Gamma** — contrast curve shape, typically 2.2 (sRGB) or 2.4 (Rec.709 for broadcast)
- **Rec.709** — HD TV color space (most YouTube / web content)
- **Rec.2020** — UHD/HDR color space
- **sRGB** — standard web color space, very similar to Rec.709 but used for images/web
- **DCI-P3** — digital cinema color space
- **Node** — a grading operation in DaVinci Resolve (can chain them)
- **Scope** — technical display of color/luma (waveform, vectorscope, histogram, parade)

## Transitions / effects

- **Hard cut** — straight cut, no effect
- **Dissolve / cross-dissolve** — fade between two clips overlapping
- **Dip to black / white** — fade to color then to next clip
- **Wipe** — one clip pushes the other out (direction and pattern vary)
- **Iris** — circular reveal
- **Whip pan** — fast camera motion blur transition
- **Flash cut / flash frame** — single-frame white/color insert
- **Speed ramp** — variable speed within a clip (slow → fast or fast → slow)
- **Freeze frame** — hold on a single frame
- **Slow-mo / slo-mo** — slower than normal playback
- **Fast-mo / timelapse** — faster than normal playback
- **Chromatic aberration** — lens artifact causing color fringing (used as style)
- **Film grain** — texture overlay mimicking film stock
- **Halation** — red bloom around highlights, mimicking film
- **Glow** — soft light bloom effect
- **Vignette** — darkening around frame edges

## Format / technical

- **Frame rate (fps)** — frames per second (24, 25, 30, 60 common)
- **Resolution** — pixel dimensions (1920x1080, 3840x2160)
- **Aspect ratio** — width:height (16:9, 9:16, 1:1, 2.39:1)
- **Codec** — encoding format (H.264, H.265/HEVC, ProRes, DNx, AV1)
- **Bitrate** — data per second, higher = better quality, larger file
- **Container** — file format wrapper (MP4, MOV, MKV)
- **Progressive / interlaced** — scan type (p / i)
- **Chroma subsampling** — 4:4:4, 4:2:2, 4:2:0 (color detail levels)
- **Bit depth** — 8-bit, 10-bit, 12-bit (gradations of color)
- **Color space / color primaries** — Rec.709, Rec.2020, etc.

## Multifandom-specific

- **Multifandom** — edit combining multiple fandoms (this whole project)
- **AMV** — Anime Music Video
- **Fanvid / vid** — umbrella term for fan-made video edits
- **Re-trailer** — fan-made trailer using official footage
- **What-if / AU** — Alternate Universe edits, remixing characters into new contexts
- **Parallel** — narrative mode, same beat told across fandoms
- **Convergent** — narrative mode, fandoms converge to single moment
- **Relay** — narrative mode, fandoms take turns per act
- **Woven** — narrative mode, fandoms interleave throughout
- **Beat sync** — cuts matching music beats
- **Lyric sync** — cuts matching specific lyrics
- **Hype edit** — high-energy, action-focused edit
- **Emotional edit** — sad / reflective / character-study edit
- **Drop** — song's highest-intensity moment
- **Buildup** — rising-tension section before a drop
- **Breakdown** — quiet/minimal section in a song
- **Bar** — one measure of music (4 beats in 4/4 time)
- **Downbeat** — beat 1 of a bar (strongest)
- **BPM** — Beats Per Minute (tempo)

## NLE-specific

- **NLE** — Non-Linear Editor (software category: Premiere, Resolve, etc.)
- **Project** — the whole editing session in your NLE
- **Sequence / timeline** — the edited result
- **Bin / folder** — organizational container for clips
- **Clip** — a piece of media in your project
- **Subclip** — a portion of a clip saved as its own clip
- **Marker** — timeline label (for beats, notes, etc.)
- **Offline / online** — offline = low-res proxies, online = full quality
- **Conform** — matching an offline edit back to full-res online media
- **Render / export** — producing final output file
- **Proxy** — low-res version for smooth editing on slow machines
